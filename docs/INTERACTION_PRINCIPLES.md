# Rent With Heldy
# Interaction Principles

Version 1.0

---

# Purpose

This document defines how the website behaves.

Not what it looks like.

Not what it says.

How it feels to use.

Every interaction should reinforce confidence, trust, and hospitality.

The experience should feel inevitable.

Nothing should surprise the user.

Nothing should distract them.

---

# Core Principles

Every interaction should feel:

• Immediate
• Predictable
• Refined
• Calm
• Intentional

Never:

• Bouncy
• Playful
• Flashy
• Hyperactive
• Distracting

---

# Performance First

Smoothness is more important than complexity.

A simple 60fps fade is better than a complicated animation.

Never sacrifice responsiveness for visual effects.

---

# Motion Philosophy

Motion exists for four reasons:

1. Explain

Show where something came from.

2. Guide

Lead attention.

3. Confirm

Reassure users that an action happened.

4. Delight

Only after the first three.

---

# Motion Timing

Small interactions

150–180ms

Cards

180–220ms

Buttons

140–180ms

Dropdowns

200–240ms

Dialogs

240–320ms

Page transitions

300–400ms

Never exceed 500ms.

Nothing should feel slow.

---

# Easing

Use smooth easing.

Avoid dramatic acceleration.

Everything should ease naturally.

Think Apple.

Not PowerPoint.

---

# Hover Behavior

Hover should never surprise.

Hover should communicate:

"This can be clicked."

Preferred hover effects:

• subtle elevation
• border emphasis
• shadow increase
• slight image scale (1.02 max)
• opacity adjustment

Never:

• rotate
• bounce
• spin
• wobble

---

# Buttons

Buttons should feel physical.

Hover

Slight lift.

Shadow increases.

Background subtly brightens.

Press

Translate down 1px.

Shadow softens.

Release

Returns naturally.

Never bounce.

---

# Links

Links should feel elegant.

Underline animations are acceptable.

Color transitions should be subtle.

Never flash.

Never pulse.

---

# Cards

Cards should not dance.

Hover should communicate quality.

Allowed:

Soft elevation

Border enhancement

Image zoom (1–2%)

Small translateY

Not allowed:

Large movement

Rotation

Perspective

Elastic motion

---

# Navigation

Navigation should feel invisible.

Dropdowns

Fade

Small vertical movement

No scaling.

Sticky navigation

Transition background smoothly.

Never abruptly appear.

---

# Hero

The hero should introduce the brand.

Text should reveal first.

Then imagery.

Then CTAs.

Never animate everything simultaneously.

---

# Scroll

Scrolling should reveal.

Not perform.

Avoid scroll hijacking.

Avoid parallax unless extremely subtle.

Each section should gently appear.

Nothing should fly in.

---

# Reveal Animations

Maximum movement:

16–24px

Maximum opacity transition:

0 → 1

Duration:

200–350ms

Stagger only where it improves readability.

---

# Loading States

Never use spinners when skeletons are possible.

Buttons

Show loading state.

Prevent double-clicks.

Maintain width.

Images

Fade in naturally.

Avoid popping.

---

# Forms

Typing should feel immediate.

Validation should happen politely.

Never punish users.

Errors should explain:

What happened.

Why.

How to fix it.

---

# Mobile

Mobile is not desktop.

Increase spacing.

Increase touch targets.

Avoid hover-dependent interactions.

Animations should be shorter.

---

# Accessibility

Respect prefers-reduced-motion.

Maintain visible focus states.

Never animate essential content out of view.

Every interaction should remain usable without animation.

---

# Spacing Rhythm

Whitespace is part of the interaction.

Do not compress sections simply to reduce scrolling.

Premium websites breathe.

Preferred spacing:

Hero → next section

120–180px

Between major sections

96–160px

Within cards

24–40px

Within text

16–24px

---

# Visual Rhythm

Alternate section density.

Example:

Editorial

↓

Image

↓

Grid

↓

Editorial

↓

Interactive

↓

Fleet

↓

CTA

Avoid repeating identical layouts.

---

# Icons

Icons support meaning.

They do not replace typography.

Reduce icon size before reducing whitespace.

---

# Shadows

Use shadows sparingly.

Increase shadows only on interaction.

Static shadows should remain subtle.

Avoid layered heavy shadows.

---

# Borders

Prefer soft borders over strong shadows.

Border contrast should increase on hover.

Never use harsh outlines.

---

# Images

Images should feel editorial.

Never stretch.

Never crop faces awkwardly.

Maintain consistent corner radius.

Use subtle fade-ins.

Avoid dramatic zoom effects.

---

# Sound

Never auto-play audio.

Ever.

---

# Delight

Delight comes from polish.

Not novelty.

The user should rarely notice the interaction.

They should simply feel:

"This website is incredibly nice."

---

# Final Test

Before shipping any interaction ask:

Does this improve understanding?

Does this improve confidence?

Does this improve trust?

Would Apple keep it?

Would Airbnb keep it?

Would Stripe keep it?

If not...

remove it.

---

# Golden Rule

When in doubt:

Use less animation.

More whitespace.

Better typography.

Better photography.

Less interface.

More hospitality
